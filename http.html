Accessing Simple HTML Webpage

To Access a simple HTML webpage, we need a Web Server, which is a computer which contains all the software and websites components. And it is connected to the internet. Also, a web server is configured with some predefined rules known as Protocols. For the access of HTML pages it requires a HTTP Server ( Software which understands URLs and HTTP protocols ). The most commonly used Web Server is Dynamic Web Server as it updates the hosted files before sending them.

When you try to access a HTML page the Browser Follows certain steps:

  • DNS Lookup : Server looks at the domain name of the website and finds the accurate website using the IP address.
  • Browser Sends Request: After a browser has performed the DNS lookup, it sends an HTTP request to the appropriate server.
  • Server Responds: Once the server receives a request, it will respond depending upon the various factors. If the file no longer exists it will give us the "404" Error.
  • Browser Renders Page: Once browser recieves the HTML page, it will be rendered so that the user can access it easily.
